# 🤖 AI 기반 회의 어시스턴트 서비스 (MVP)
> **Django를 활용한 회의록 데이터 관리(CRUD) 시스템 구축 및 실시간 AI 요약 서비스 협업 프로젝트**

---

## 🎯 1. 프로젝트 개요
본 프로젝트는 반복적이고 비효율적인 회의록 작성 프로세스를 개선하기 위해 개발된 **AI 기반 회의 보조 서비스**입니다. 
팀원 간의 원활한 협업을 통해, 실시간으로 입력되는 회의 음성/텍스트 데이터를 분석하여 핵심 안건과 주요 이슈를 구조화된 요약본으로 제공하고, 이를 데이터베이스에 안정적으로 영속화하는 웹 서비스를 구축했습니다.

---

## 🛠️ 2. 기술 스택 (Tech Stack)

### Backend / Framework
* **Python 3.x**
* **Django 4.x / 5.x**

### DB & Environment
* **SQLite / MySQL** (로컬 개발 및 데이터 모델링)
* **python-dotenv** (보안 환경변수 관리)

---

## 📂 3. 프로젝트 구조 (Directory Structure)

본 프로젝트는 확장성과 유지보수성을 고려하여 **AI 모듈**, **Backend (Django)**, **Frontend** 영역을 독립적인 레이어로 분리하여 설계했습니다.

```text
AIMeetingAssistant/
│
├── AI/                     # 🤖 AI 핵심 로직 및 모델 연동 모듈 (팀원 담당)
│
├── New_KT-BE/              # ⚙️ Backend (Django Framework)
│   ├── manage.py
│   ├── core/               # 프로젝트 메인 설정 (settings.py, urls.py 등)
│   └── meeting/            # 📌 회의록 관리 CRUD 및 비즈니스 로직 앱 (본인 전담)
│       ├── models.py       # 회의록 저장 및 조회를 위한 DB 스키마 설계
│       ├── views.py        # 게시판 CRUD 비즈니스 로직 구현
│       └── urls.py         # RESTful API 엔드포인트 URL 매핑
│
└── New_KT-FE/              # 💻 Frontend (사용자 웹 인터페이스)
```
## 🛠️ 4. 담당 역할 및 핵심 기여도 (Backend CRUD 전담)
본 프로젝트에서 저는 백엔드(Backend)의 핵심 근간이 되는 데이터 모델링 및 게시판 CRUD 비즈니스 로직 개발을 전담하였으며, 팀원의 AI/웹소켓 모듈이 안정적으로 결합될 수 있도록 데이터 파이프라인의 기반을 닦았습니다.

1) 회의록 데이터 아키텍처 설계 및 CRUD 구현
데이터 모델링 (models.py): 회의 ID, 제목, 원본 텍스트, AI 최종 요약본, 생성 시간 등을 구조화하여 저장할 수 있는 데이터베이스 스키마를 직접 설계했습니다.

게시판 비즈니스 로직 (views.py): 사용자가 작성한 회의록을 안전하게 저장(Create)하고, 과거 회의록 목록 및 상세 내용을 안정적으로 조회(Read)할 수 있는 웹 서비스의 핵심 기능을 구현했습니다.

2) RESTful URL 매핑 및 연동 인터페이스 구축
클라이언트(프론트엔드) 및 웹소켓 데이터 흐름을 고려하여 직관적이고 체계적인 API 엔드포인트를 설계하고 URL 매핑(urls.py)을 수행했습니다.

팀원이 구현한 웹소켓 커널(consumers.py)에서 AI(OpenAI) 연동을 통해 추출한 최종 회의 요약 데이터를 내가 설계한 모델 아키텍처(Meeting.objects.create)에 유기적으로 연결 및 저장되도록 조율했습니다.

3) 환경변수 분리 및 소스코드 보안 강화 (.env)
API 인증 키(OpenAI), DB 접속 정보 등 민감한 자격 증명이 오픈소스 저장소에 노출되지 않도록 python-dotenv를 도입하고 시스템 환경변수(.env)로 격리 관리했습니다.

.gitignore 설정을 적용하여 무단 보안 유출 취약점을 사전에 전면 차단했습니다.
